Implementation Notes

Issue #40 — 扩展 enrich 包支持 IPv6 (Cymru origin6 + ipinfo v6) — 2026-06-30

Design Decisions

Decision Branch on address family inside the existing Lookup loop

Ambiguity: The spec required v6 support without regressing v4, but did not prescribe whether to fork a separate v6 path.

Choice: Kept a single Lookup loop; per-IP it picks the zone via ip.To4() (origin) vs. else (origin6), reusing the same goroutine/semaphore/AS-name cache machinery.

Rationale: One concurrency path means no duplicated bounding logic and the AS-name cache is shared across families. Minimal surface area, no regression risk to v4.

Decision reverseV6Nibbles emits low-then-high nibble per byte, full 32 nibbles

Ambiguity: Cymru's origin6 zone expects ip6.arpa-style reversed nibbles; the exact byte/nibble ordering is easy to get wrong.

Choice: Iterate bytes high-index to low, emitting lo.hi per byte, pinned by a unit test against 2001:4860:4860::8888 (31 dots, 32 nibbles).

Rationale: A known fixture locks the wire format so a future refactor can't silently break ASN resolution.

Deviations

Deviation ipinfo provider needed no code change for v6

Spec said: Confirm IPInfoProvider works for v6 and sends the v6 literal in the request path.

Implemented instead: No production change — the existing url.PathEscape(ip.String()) already emits the v6 literal. Added a httptest-based test asserting the literal appears in the path rather than modifying the provider.

Why: The code was already correct; the spec item was a verification requirement, satisfied by a test.

Tradeoffs

Tradeoff Skip unusable IPs silently vs. surface an error

Alternatives: (a) continue past any IP that is neither v4 nor v16; (b) return an error for malformed input.

Chosen: Silent continue, matching the existing best-effort enrichment contract where missing data degrades gracefully rather than aborting a whole trace.
