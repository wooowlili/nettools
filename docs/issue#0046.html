Implementation Notes

Issue #46 — 创建 cmd/traceroute6 CLI 并接线 flag/provider — 2026-06-30

Design Decisions

Decision Mirror cmd/traceroute flag set verbatim, swap only TOS→Traffic Class semantics

Ambiguity: The AC (US-007) says "aligned flags" but does not enumerate which flags carry v6-specific wording.

Choice: Kept every flag name, shorthand, and default identical to cmd/traceroute. The only semantic change is -t/--tos, whose help text now reads "IPv6 Traffic Class value (v6 analogue of TOS/DSCP)" and feeds Config.TrafficClass rather than Config.TOS. -m/--max-hops wording gained "(Hop Limit)" and maps to Config.HopLimit.

Rationale: Output/UX parity with the v4 tool was the explicit project goal (US choice 4A). Users switching between traceroute and traceroute6 get a zero-surprise flag surface; only the underlying header field differs, which is invisible at the CLI.

Decision Resolve hostnames to the first AAAA address

Ambiguity: A hostname may have both A and AAAA records; the spec doesn't say which to pick.

Choice: resolveTargets iterates net.LookupHost results and picks the first address where ip.To4()==nil && ip.To16()!=nil; IPv4-only inputs yield "not an IPv6 target (use traceroute for IPv4)".

Rationale: Matches the v4 tool's "first matching family" behavior. The clear error message steers IPv4 users to the correct binary rather than silently failing.

Deviations

Deviation IPv6-literal error wording differs from v4

Spec: The v4 tool rejects v6 input with "IPv6 target not supported yet".

Implemented: The v6 tool rejects v4 input with "not an IPv6 target (use traceroute for IPv4): <in>" and rejects no-AAAA hostnames with "no IPv6 (AAAA) address for <host>".

Why: The two tools are mirror opposites; each should point the user at its sibling. "not supported yet" would be misleading in the v6 binary, which fully supports v6.

Tradeoffs

Tradeoff Duplicate main.go vs shared CLI helper

Alternatives: (a) copy cmd/traceroute/main.go and adapt; (b) extract a shared flag-parsing/CLI helper used by both binaries.

Chosen: (a) — an independent cmd/traceroute6/main.go.

Why: The project convention is fully independent paired packages (ping/ping6, sonar/sonar6, traceroute/traceroute6). A shared CLI helper would couple the two Config types (different field names: TOS vs TrafficClass, MaxHops vs HopLimit) and break that boundary for marginal dedup gain.
