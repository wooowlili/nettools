Implementation Notes

Issue #43 — 用 goscapy 构造 v6 探针并匹配回包 — 2026-06-30

Design Decisions

Decision Skip the 4-byte ICMPv6 error preamble before the quoted packet

Ambiguity: The spec said to parse the embedded original IPv6 header (inner dst at [24:40]) but did not mention the framing that precedes it inside the ICMPv6 error body.

Choice: matchEmbedded drops the first 4 bytes of the Raw load (the ICMPv6 error's unused/MTU word) before reading the 40-byte inner IPv6 header.

Rationale: Time Exceeded (type 3) and Dest Unreachable (type 1) both carry a 4-byte field after the base header, then the invoking packet. goscapy leaves those bytes in Raw; a debug dump of a synthesized reply confirmed the inner header starts at offset 4, not 0.

Decision Match Echo Reply identity on the "ICMPv6 Echo Reply" sub-layer

Ambiguity: goscapy splits ICMPv6 into a base layer (type/code/chksum) and a typed sub-layer; the spec only referenced "id/seq".

Choice: echoIdentMatches reads id/seq from the sent ICMPv6 Echo layer and the received ICMPv6 Echo Reply layer (distinct layer names per goscapy's type→layer binding 128/129).

Rationale: Mirrors the v4 misattribution guard: a single Echo Reply must only match the probe whose id+seq it echoes, else it would attach to every concurrent in-flight probe.

Deviations

Deviation Inner-header parse uses a fixed 40-byte offset, no IHL math

Spec/v4 parallel: The v4 matchEmbedded computes header length from the IHL nibble (load[0]&0x0F * 4) and reads inner dst at [16:20].

Implemented: The IPv6 header is fixed at 40 bytes, so the v6 version drops IHL logic, reads inner dst at [24:40], and slices the upper-layer header at [40:].

Why: IPv6 has no IHL field. This is the intended v6 adaptation, not a regression.

Tradeoffs

Tradeoff Push the Echo body separately vs a single Over() chain

Alternatives: (a) ip.Over(icmp) then pkt.Push(echo); (b) a single chained constructor.

Chosen: Build the base ICMPv6 layer, then Push the Echo body — matching how goscapy registers 128→"ICMPv6 Echo" as a next-layer and how the checksum build hook locates the IPv6 layer below.

Why: Keeps the layer stack explicit and consistent across the ICMP/UDP/TCP branches (UDP also pushes a Raw payload the same way).

Open Questions

Question Quoted-payload length on real routers

Assumption: Real ICMPv6 errors quote at least the 40-byte inner IPv6 header plus the leading L4 bytes (8 for id/seq or ports). The synthesized test reply quotes the full packet.

Verify: On live traces, confirm matching still succeeds against routers that quote the RFC 4443 minimum (as much as fits in the min-MTU 1280 packet — effectively always enough here). Low risk, but worth a real-network sanity check during #44/#46 integration.
